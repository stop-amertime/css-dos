# 2026-07-13 - Windows 1.01 cart: boots to the MS-DOS Executive, apps launch

`carts/windows101` runs real Windows 1.01 (owner's retail 5.25" floppies) on
the msdos4 boot path, CGA mode 6, 1.2M ROM floppy, 293.5 MB cabinet.
Install: retail SETUP.EXE run under scripted DOSBox-X (all 5 disks merged
into one dir mounted as A: - SETUP's per-disk label checks all pass, no
swapping; AUTOTYPE with alternating `c n` tail since SETUP flushes type-ahead
at each prompt) → genuine WIN.COM + WIN100.BIN/OVL fastboot blobs, which
loose disk files can't provide (retail kernel has no slow-boot).
Key finding: Win 1.01 boots on MS-DOS 4.00 but its app loader **gates on
INT 21h AH=30h returning 2.x/3.x** - internals are fine (Executive runs);
DOSBox A/B with only the reported version flipped proves it. Fix:
`SETVER3.COM`, 51-byte TSR in the cart (source alongside), answers 3.30;
AUTOEXEC loads it before WIN. Verified: fast-shoot logo ~2M ticks, Executive
~8.5M, CLOCK.EXE launched via injected keys (Down×6+Enter); web path PASS
(vendored wasm, compile 12.9s, mode 6 + ink at 31.8s wall). New gate:
`node tests/harness/run.mjs windows` (Executive + key-launch phash diff).
fast-shoot grew `--press-events` (forwarded to calcite-cli); web-boot grew
`--gfx-sentinel=MODE` for graphics-mode cabinets.
